# Core

Base reusable para arrancar proyectos web de forma consistente antes de disenar, maquetar o publicar una preview.

## Archivos

- `web-project-questionnaire.md`
  Cuestionario corto y ordenado para arrancar cualquier proyecto o preview.
- `web-project-intake.md`
  Sistema de preguntas, decisiones y convenciones para definir una web antes de crearla.
- `web-project-brief-template.md`
  Plantilla lista para duplicar y rellenar en cada nuevo proyecto.

## Uso recomendado

1. Pasar `web-project-questionnaire.md`.
2. Duplicar `web-project-brief-template.md`.
3. Rellenar las respuestas con el cliente o internamente.
4. Elegir el modelo: `Lite`, `Express`, `Personalizada`, `Web Pro` o `App`.
5. Definir hero, estilo, contenido y requisitos tecnicos.
6. Crear la preview con una ruta clara, por ejemplo:
   - `portfolio/{cliente}/weblite/`
   - `portfolio/{cliente}/weexpress/`
   - `portfolio/{cliente}/webpro/`

## Patrones destacados

- `Modelo Clueso`
  Portada dominante con imagen persistente durante el inicio del scroll y contenido entrando por encima sin split ni sticky visible.

## Objetivo

Que cada nueva propuesta nazca con el mismo nivel de claridad:

- que tipo de web se esta construyendo
- que direccion visual tiene
- que contenido necesita
- que extras incluye
- que se publica como preview
