# Web Project Intake System

Sistema base para iniciar cualquier proyecto web o preview de portfolio sin improvisar.

## 1. Modelo del proyecto

Primera pregunta obligatoria:

- `Lite`
  Monopage ligera, muy enfocada, ideal para propuesta rapida o presencia premium minima.
- `Express`
  Web multipage clara, con secciones internas definidas y estructura comercial.
- `Personalizada`
  Direccion visual a medida cuando `Lite` y `Express` se quedan cortas.
- `Web Pro`
  Proyecto mas completo con estrategia, arquitectura, SEO y bloques avanzados.
- `App`
  Producto con flujos, paneles, logica o experiencia mas cercana a software.

## 2. Tipo de hero

Definir el comportamiento del primer pantallazo antes de tocar layout.

Opciones base:

- `Hero fijo Lite`
  Hero limpio, directo y sin exceso de capas.
- `Modelo Clueso`
  Portada dominante con persistencia visual invisible: la imagen se mantiene presente al bajar y el contenido entra por encima como continuidad editorial.
- `Full-screen reveal`
  Portada a pantalla completa que da paso editorial al contenido al hacer scroll.
- `Split premium`
  Copy y visual en equilibrio, muy controlado.
- `Cards commercial`
  Hero con estructura de tarjetas o bloques definidos.
- `Editorial minimal`
  Hero con mucho aire, tipografia y fotografia dominante.
- `Video / motion light`
  Movimiento suave sin convertir la experiencia en algo pesado.

## 3. Direccion visual

Elegir una sola direccion principal para evitar mezclas flojas.

Ejemplos:

- `Clueso editorial`
- `Premium minimalista`
- `Editorial`
- `Nike / Porsche agresivo limpio`
- `Blog de viajes`
- `Tienda generica`
- `Marca personal elegante`
- `Wellness / belleza`
- `Tecnologica / producto`

## 4. Arquitectura base

Preguntas minimas:

- `Monopage o multipage`
- `Numero de paginas`
- `Paginas clave`
  Ejemplos: `inicio`, `servicios`, `entrenos`, `online`, `contacto`, `legal`
- `Blog o sin blog`
- `Eventos o sin eventos`
- `Tienda o sin tienda`
- `Landing de captacion o web completa`

## 5. Contenido y conversion

Definir que hace la web y que accion queremos provocar.

Preguntas:

- `CTA principal`
  Ejemplos: `agendar primera cita`, `pedir presupuesto`, `escribirme por WhatsApp`
- `CTA secundario`
- `Metodo de contacto`
  Ejemplos: `WhatsApp`, `formulario`, `email`, `telefono`, `Calendly`
- `Prueba social`
  Testimonios, logos, casos, antes/despues, reseñas.
- `Servicios clave`
- `Oferta destacada`
- `Tono de copy`
  Ejemplos: `directo`, `premium`, `cercano`, `editorial`, `tecnico`

## 6. Requisitos funcionales

Checklist rapido:

- `Banner cookies`
  `Personalizado` o `estandar`
- `Legal`
  `aviso legal`, `privacidad`, `cookies`
- `Idiomas`
- `SEO local`
- `Formularios`
- `Integraciones`
  Ejemplos: `Calendly`, `Stripe`, `newsletter`, `maps`
- `Analitica`
- `Velocidad prioritaria`

## 7. Assets disponibles

Antes de disenar, confirmar material real.

- `Logo`
- `Fotos del cliente`
- `Brand colors`
- `Tipografia`
- `Textos base`
- `Referencias visuales`
- `Dominio o slug de preview`

## 8. Sistema de preview

Toda propuesta debe nacer con una ruta clara.

- `Ruta de preview`
  Ejemplo: `portfolio/indiralopez/weblite/`
- `Nombre visible de la propuesta`
- `Version`
  Ejemplo: `v1`, `v2`, `cliente-review`
- `Estado`
  `borrador`, `review interna`, `lista para cliente`, `publicada`

## 9. Reglas para decidir rapido

- Si el cliente necesita algo simple, visual y muy enfocado: `Lite`
- Si necesita paginas internas claras: `Express`
- Si la referencia visual manda mucho: `Personalizada`
- Si hay estrategia, SEO, blog y estructura mayor: `Web Pro`
- Si hay logica de producto o panel: `App`

## 10. Patrones que conviene guardar

Cada vez que una solucion funcione bien, registrar el patron.

Ejemplos:

- `Hero fijo - Lite`
- `Modelo Clueso - Lite`
- `Full-screen reveal - Lite`
- `Split premium - Express`
- `Cards afiladas - Express`
- `Editorial portfolio hero`
- `CTA WhatsApp premium`

La idea es convertir estos patrones en biblioteca interna para futuras propuestas.
